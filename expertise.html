QR codes

Le code QR est un code-barres à deux dimensions (data matrix) constitué de modules noirs disposés dans un carré à fond blanc.
Il peut stocker jusqu'à 4 296 caractères alphanumériques, avec un système de redondance pour la correction d'erreur.
Pour lire un code QR, l'utilisateur de smartphone peut utiliser:
  • QR Reader sur iPhone
  • Barcode Scanner sur Android

Il peut contenir :

  • du texte
  • l'adresse d'un site internet
  • la carte de visite d'une personne (vCard, MeCard)
  • un numéro de téléphone
  • un rendez-vous ou un événement (iCalendar)
  • les informations pour se connecter à un réseau Wifi
  • le lien pour télécharger une application
  • une localisation
  • ...
Moventes intègre la lecture de QR codes dans ses applications pour importer facilement des informations, ou pour promouvoir leur téléchargement.

mBaaS

Une solution mobile se compose de:
  • un front-end, une application mobile installée sur le smartphone des utilisateurs
  • un back-end composé d'une base de données et de scripts serveur avec lesquels l'application mobile intéragit
La plateforme mBaaS (mobile Back-end as a Service) fournit des "briques" facilement assemblables pour constituer le back-end rapidement, ainsi que l'hébergement de celui-ci dans le cloud (public ou privé) pour toute la durée de vie de l'application.
  • Fonctionnalités d'une plateforme mBaaS
  • stockage et gestion des données
  • identification et gestion des utilisateurs
  • accès aux autres données existantes de l'entreprise
  • envoi de notifications aux utilisateurs de l'application
  • statistiques d’utilisation de l'application
  • synchronisations entre le smartphone et le serveur
  • sécurisation des échanges de données

Paiement mobile

In App Purchase (IAP)
Pour les achats in-apps, le store d'applications sert d'intermédiaire (en prélevant une commission de 30% sur les transactions) pour débiter le compte bancaire de l’utilisateur.

Ce mode de paiement est imposé pour tous les achats permettant de débloquer des fonctionnalités (ou acquérir des "biens immatériels") de cette application mobile.
Porte-monnaie électronique
Le portefeuille électronique (digital wallet) permet de faire des achats en ligne (via un compte sécurisé) ou en magasin (via un téléphone, par NFC, Bluetooth ou SMS).
Il existe de nombreuses solutions, telles que :
  • PayPal
  • Google Wallet
  • M-Pesa
Carte bancaire
Les achats peuvent être faits directement par carte bancaire, en recopiant son numéro à 16 chiffres et son cryptogramme.
Seuls les PSP respectant les normes de sécurité PCI DSS sont autorisés à traiter et stocker ces informations de carte bancaire.
Un contrat VAD est fourni par un établissement acquéreur pour pouvoir encaisser les transactions.
Moventes sélectionne pour chaque application mobile la méthode de paiement la plus adaptée aux besoins des utilisateurs, grâce à une analyse détaillée des coûts et contraintes des différentes solutions technologiques existantes.
